# Introducción

Este documento describe el plan del proyecto para el desarrollo de una **aplicación de escritorio nativa instalable orientada a la gestión de asistencia y control operativo biométrico**, diseñada como parte del portafolio de **RAMar Software Studio** y con potencial de implementación comercial en computadoras de **micro y pequeñas empresas (MYPES)**.

El objetivo principal del proyecto es ofrecer una **solución de escritorio eficiente, de alto rendimiento y segura off-line**, que funcione como quiosco local sin depender de servicios externos web, asegurando latencia nula en interacción por cámara.

El sistema está pensado para cubrir necesidades reales de las MYPES, priorizando **simplicidad, rapidez de operación y control administrativo**, aspectos clave en entornos con recursos limitados y procesos manuales.

El desarrollo se centrará en la construcción de un **Producto Mínimo Viable (MVP)** que demuestre la viabilidad técnica, funcional y comercial del sistema, sirviendo tanto como caso de estudio para el portafolio de **RAMar Software Studio** como base para futuras mejoras y escalabilidad.

---

## Objetivos del Proyecto

- Diseñar y desarrollar un sistema de escritorio WPF nativo aplicable a entornos reales de MYPES con hardware fotográfico estándar.
- Reducir la dependencia de procesos manuales mediante automatización básica y control centralizado.
- Ofrecer una solución que no requiera conexión a internet ni hardware especializado.
- Presentar un producto sólido y documentado como parte del portafolio de RAMar Software Studio.
- Sentar las bases para una posible comercialización y personalización del sistema según el cliente.

---

## Enfoque del Producto

El sistema se desarrolla bajo un enfoque **práctico y orientado al negocio**, priorizando:
- Implementación local dentro de la infraestructura existente del cliente.
- Acceso mediante clientes pesados (.exe interactivo rápido).
- Facilidad de instalación, uso y mantenimiento.
- Costos de adopción bajos para micro y pequeñas empresas.

Este enfoque permite que el producto sea atractivo tanto para demostraciones técnicas como para posibles clientes interesados en una solución simple y funcional.

---

## Naturaleza del Proyecto

El proyecto tiene un **doble enfoque**:
- **Profesional**, al formar parte del portafolio técnico de RAMar Software Studio.
- **Comercial potencial**, al estar diseñado para responder a necesidades reales de MYPES.

Aunque su desarrollo inicial se realiza en un contexto académico y formativo, el sistema no se limita a fines educativos, sino que busca demostrar capacidades reales de análisis, diseño y desarrollo de software aplicable al mercado.

---

## Presupuesto y Recursos

El desarrollo del sistema se realiza sin un presupuesto económico directo, utilizando:
- Equipos personales del equipo de desarrollo.
- Herramientas de desarrollo y tecnologías de uso libre o gratuito.
- Infraestructura local para pruebas y validaciones.

Este modelo refleja un escenario realista para MYPES, donde la optimización de recursos es un factor clave.

---

## Tiempo Estimado

El desarrollo del MVP está planificado para ejecutarse en un período aproximado de **un mes**, considerando las etapas de análisis, diseño, implementación y pruebas, alineadas tanto a los objetivos académicos como a la presentación del producto en el portafolio.

---

## Limitaciones

Durante la planificación del proyecto se han identificado las siguientes limitaciones:

- **Alcance controlado**, enfocado en funcionalidades esenciales para el MVP.
- **Equipo de desarrollo reducido**, lo que limita la implementación simultánea de múltiples módulos.
- **Priorización de estabilidad y simplicidad** sobre características avanzadas.

Estas limitaciones han permitido definir un alcance realista, asegurando la entrega de un producto funcional, demostrable y con valor potencial para micro y pequeñas empresas.
