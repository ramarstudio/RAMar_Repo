# 📚 Metodología de Planeamiento (SDLC)

En RAMar Studio aplicamos un ciclo de vida de desarrollo de software (SDLC) riguroso para asegurar la máxima calidad corporativa antes de tirar una sola línea  de código.

---

## Fases de Producción de Software

### 1. Identificación de Casos de Uso
Traducimos las necesidades operativas de la empresa en historias técnicas y medibles. Se modela qué debe hacer el administrador frente a las limitaciones de un usuario raso.

### 2. Matriz MoSCoW Estricta
Previo a cualquier presupuesto, priorizamos el alcance dividiendo los módulos en *Must* (Obligatorio), *Should* (Recomendado), *Could* (Opcional) y *Won't* (Descartado para el MVP). Esto previene desviaciones de tiempo y sobrecostos por "Feature Creep" (añadir mil funciones inútiles).

### 3. Decisión de Arquitectura Truncada (ADRs)
Se evalúan objetivamente todas las alternativas: ¿Web? ¿Escritorio? ¿Móvil? ¿Nube?. Luego se **justifica por escrito** (Architecture Decision Record) el pivoteaje al ecosistema técnico final. Esto previene que el equipo programe en tecnología caprichosa y se adhiera a algo justificado algorítmicamente.

### 4. Diseño UX / UML Acoplado
Se trazan los diagramas de interacción estructural y los bocetos visuales que garantizan que el operador humano no experimentará fricción al usar el sistema resultante.
